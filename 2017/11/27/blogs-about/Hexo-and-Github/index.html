基于Hexo和Github搭建属于自己的博客

发表于 2017-11-27 | 分类于 博客搭建教程

写在前面

应小松学长的请求，整理一下Hexo + Github 搭建博客的步骤。
我自己都还算不上半吊子，整理的东西也参差不齐，希望不要介意。





搭建环境的准备

一、配置Node.js环境

（1）下载Node.js安装文件

Windows Installer 32-bit
Windows Installer 64-bit

保持默认设置，直至安装完成。

（2）检查安装

打开Windows运行界面（同时按Win+R键)；

点击确定
在命令行界面输入：

node -v
npm -v

结果如下图所示，则安装成功




二、配置Git环境

（1）下载Git安装文件

Git官网下载地址

全部保持默认。

（2）检查安装

同样打开Windows运行界面
在命令行界面输入：

git - -version

结果如下图所示，则安装成功






Github账号的注册和配置

（1）Github的注册

打开Github官网
填写个人信息，用户名、邮箱和密码，
填写用户名时慎重点，它直接和博客地址挂钩
然后进入所填邮箱确认注册。

（2）创建代码库

登陆成功后，点击页面右上角加号，选择New repository

在Repository name下填写yourname.github.io，这是标准格式
并且yourname必须和用户名一致
勾选创建README文件


（3）代码库设置

成功创建后：

点击界面右侧的Settings，向下拖动，直到看见GitHub Pages

再点击Automatic page generator，Github将会自动创建出一个gh-pages的页面。
如果以上配置没有问题，那么yourname.github.io这个网址就可以正常访问了。





SSH配置

设置Git的user name和email：

git config - -global user.name “name”
git config - -global user.email “email@example.com“


生成SSH Key

执行以下命令：

ssh-keygen -t rsa -C “your_email@example.com“

一直回车直至密钥生成成功。
找到密匙所在目录为/c/Users/you_PC/.ssh/id_rsa,pub，复制密钥。

添加密钥

在Github页面右上角加号旁边的图标，找到settings并点击（区别于仓库的settings）

点击SSH and GPGkeys选项，选择New SSH key

将刚才负责的密钥粘贴后点击Add SSH key完成

测试是否成功

输入以下命令：

ssh -T git@github.com

若出现continue connecting (yes/no)? 输入yes。





Hexo

安装Hexo

找一个合适的地方新建文件夹hexo，如（D盘->Blogs->hexo）
直接在桌面空白处点击鼠标右键，打开Git Bash Here，并进入该目录（以后的命令输入都是这样）

分别输入

npm install hexo-cli -g
npm install hexo - -save

运行完之后，再输入以下命令检查是否安装成功

hexo -v

若如下图所示，则安装成功


hexo的相关配置

（1）初始化

接着上面的操作，输入：

hexo init
npm install

（2）首次体验

hexo g
hexo s


打开http://localhost:4000/

如下图所示，则hexo配置结束。






Hexo和Github page的连接

（1）配置Deployment

这里推荐一款文本编辑器，Atom
起初因为大神推荐且适配hexo方才使用，后来愈发觉得好用。

在_config.yml文件中，找到Deployment，做如下修改：

deploy:
type: git
repo: git@github.com:yourname/yourname.github.io.git
branch: master

最后执行

hexo clean
hexo g
hexo d

如果配置成功，现在访问yourname.github.io，将看到和刚才访问http://localhost:4000/
时一样的页面。





主题设置

这里有几点需要注意：

1.两个配置文件：
站点配置文件指的是博客文件根目录下（如我的是hexo目录）的_config.yml，
主题配置文件是主题文件夹下（我的是hexo->themes->next）的_config.yml

2.一些简单的hexo命令
hexo new “title” # 生成新文章，在\source_posts\title.md目录下
hexo new page “title” # 生成新的页面，后面可在主题配置文件中配置页面
hexo c # 清除缓存文件 (db.json) 和已生成的静态文件 (public)
hexo s # 本地预览，通过访问http://localhost:4000/
hexo d -g # 部署，最终显示在博客上

更多可以参考hexo官网

3.Markdown标记语言
hexo 可以兼容HTML标记语言，
但Markdown标记语言相比HTML来说更简单好用。
如何在博客文章中使用Markdown

下载主题

hexo官网主题

我选的是Next，黑白搭配，超级简洁
克隆最新版本
输入命令：

git clone https://github.com/iissnan/hexo-theme-next themes/next

完成后，打开站点配置文件， 找到theme字段，并将其值更改为next。

敲入命令：

hexo d -g

看看效果。

此后，修改完内容，可以先通过hexo s 本地预览
满意后再部署用hexo d -g 部署发布。
最后就可以访问博客yourname.github.io。

美化主题

Next 主题的美化网上可以找许多。
这里推荐一个比较完整的hexo的next主题个性化教程:打造炫酷网站





更换电脑或者重装系统

重装系统

因为我们的数据都是存放在hexo文件夹里，比如source/_ posts，所以一定要备份好hexo。
我的hexo安装在D盘，所以重装系统没有太大的影响。


重装系统后，按照上面的步骤，重新安装node.js和Git，并配置ssh就可以了。

更换电脑

我还没有换过，但是觉得应该和重装系统后的操作差不多。

Github代码仓库

上传文件指令：

git init（未初始化）
git add file_name
git commit -m “first commit”
git remote add origin address.git（仓库地址）
如果出错，则git remote rm origin后再重新进行上一步
git push -u origin master
