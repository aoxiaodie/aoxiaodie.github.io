继承

定义：从已经存在的类中定义新的类，称为继承。

父类和子类

如果一个类C1扩展自另一个类C2，那么
C2称为超类，也称为父类或基类；
C1称为次类，也称为子类、扩展类或派生类。

注意点：
1、每个类只能有一个父类，即不能多重继承。
2、父类的私有数据域不能被子类直接使用，但如果父类定义了公共的访问器/修改器（set/get方法），那么可以通过这些公共的访问器/修改器来访问或修改他们。

super关键字

调用父类的构造方法

父类的构造方法不会被子类继承，只能用super关键字调用。

1、显式调用
super()或super(parameters)
必须出现在子类构造方法的第一行。

2、隐式调用
如果没有显式调用，编译器会自动将super作为构造方法的第一条语句。
当构造一个子类的对象时，子类构造方法会在完成自己的任务之前，首先调用它的父类的构造方法，并延续下去，这也就是构造方法链。

简单来说，就是在一个子类中，如果没有显式调用父类的构造方法，那么它会先找到父类的无参构造函数执行。
一定会先调用父类的构造方法。

抽象类和接口

抽象类

注意点：
1、抽象类不能创建对象，只能作为父类；
2、抽象类的构造方法定义为protected，因为只能被子类使用；
3、抽象方法不能包括在非抽象类中，包含抽象方法的类必须是抽象的；
4、抽象类不能用new来初始化；
5、子类可以覆盖父类的方法并把它定义为abstract（@Override声明为强制覆盖），即父类是具体的，子类也可以是抽象的；
6、虽然抽象类不能用new创建对象，但可以作为一种数据类型：GeometricObject[] objects = new GeometricObject[10];。

Cloneable接口

Cloneable接口给出一个可克隆的对象

定义

一个带空体的接口称为 标记接口，用来表示一个类拥有某些特定的属性。

深复制和浅复制

Object类中的clone方法将原始对象的每个数据域复制成目标对象。
如果一个数据域是基本类型，复制的就是它的值；
如果一个数据域是对象，复制的就是该域的引用。


如果数据域是基本对象，则不需要特殊考虑；
如果数据域中有对象类型，则要使用深复制。


浅复制：数据域中有对象类型，原始对象和目标对象指向同一个地址单元；
深复制：原始对象和目标对象指向不同地址单元，即重新创建了一个对象。

接口和抽象类

类的扩展做单一继承，但允许使用接口做多重扩展。
使用关键字extends，接口可以继承其他接口，称为子接口。

通常推荐使用接口。
