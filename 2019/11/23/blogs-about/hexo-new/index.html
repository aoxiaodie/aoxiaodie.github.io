更换电脑后如何在新电脑重新搭建环境

发表于 2019-11-23 | 分类于 博客搭建教程

写在前面

一年没有更新博客了，最近又写了很多东西想要更新，结果换了公司的电脑，部署的时候出现问题，弄了两天才好。
为此，我决定写下我的踩坑步骤。





环境准备

跟搭建的时候一样，需要安装Node.js和Git，具体看之前的文章。
基于Hexo和Github搭建属于自己的博客





SSH配置

同样跟搭建时一样，另外有几个注意点：
1、user_name是Github的用户名；
user_email同。


2、直接用电脑终端进行操作。





Hexo安装配置

新建文件夹

新建一个文件夹用于存放hexo
并将原来文件里的 _config.yml、theme、source、scaffolds、package.json、.gitignore 6 个文件拷贝放入新文件夹中。

安装

使用Git bash在新建的hexo目录下操作
分别输入：

npm install hexo-cli -g
npm install hexo - -save

运行完之后，再输入以下命令检查是否安装成功

hexo -v


初始化

hexo init
npm install

此时查看hexo目录下文件已经完整了，跟原来的相同。

部署发布

hexo s # 本地预览，通过访问http://localhost:4000/
hexo d -g # 部署，最终显示在博客上






自定义主页

重新部署后发现自定义的主页消失了，在网上找了一圈，最先找到的是这个：

在hexo根目录/source文件夹下新建一个index.md即可

但我的自定义index.md一直都在，又找了许久，终于找到这个：

删除node_modules里的hexo-generator-index模块（因为这个是用来渲染主页的，自定义主页不需要）

联想一下，我们的node_modules是重新生成的，所以存在hexo-generator-index模块，这就导致自定义主页显示。
